DESCRIPCIÓN
-----------
Sistema de ciclo iterativo (Loop) para predecir jugadas de:
  - Baloto (miércoles y sábados)
  - Revancha (miércoles y sábados)
  - Miloto (lunes, martes, jueves, viernes)

Usa 10 métodos estadísticos + IA (Gemini) y se auto-ajusta
hasta alcanzar un objetivo de acierto configurable.

USO
---

A) Ejecutar el loop completo (recomendado):
   python loop.py

   Esto realiza:
     1. Scraping de los últimos 300 sorteos de baloto.com
     2. Análisis con 10 métodos
     3. Genera 2 sugerencias por juego (Baloto, Revancha, Miloto)
     4. Verifica aciertos contra resultados históricos
     5. Ajusta pesos si acierto < 90%
     6. Repite hasta 10 iteraciones o hasta alcanzar el objetivo
     7. Genera informe HTML y Excel

B) Ejecutar módulos individuales:
   python scraper.py        → Solo actualizar datos
   python analisis.py       → Solo análisis estadístico
   python sugerencias.py    → Solo generar jugadas (sin IA)
   python verificacion.py   → Solo verificar aciertos
   python informe.py        → Solo generar reporte de prueba

CRITERIO DE ACIERTO
-------------------
Una jugada se considera "hit" si tiene >= 2 coincidencias
con el resultado oficial del sorteo del mismo día.
Este parámetro se puede cambiar en config.py:
  ACIERTO_MIN_NUMS = 2   ← cambiar a 3 para criterio más estricto

CONFIGURACIÓN AVANZADA
-----------------------
Todas las opciones están en config.py:

  N_SUGERENCIAS = 2          → Máximo jugadas por juego
  OBJETIVO_ACIERTO = 0.90    → 90% de hits como objetivo
  MAX_ITERACIONES = 10       → Iteraciones máximas del loop
  ACIERTO_MIN_NUMS = 2       → Mínimo aciertos para contar como hit
  IA_MODEL_NAME = "gemini-2.0-flash"  → Modelo de IA

MÉTODOS ESTADÍSTICOS IMPLEMENTADOS
------------------------------------
M1  Frecuencia absoluta         → Números más/menos repetidos históricamente
M2  Frecuencia por posición     → Números favoritos en cada posición (N1..N5)
M3  Análisis de rachas          → Números "vencidos" (hace mucho que no salen)
M4  Suma estadística            → Rango P25-P75 de sumas históricas
M5  Correlación de pares        → Pares que salen juntos frecuentemente
M6  Análisis de paridad         → Distribución óptima par/impar
M7  Análisis por décadas        → Distribución por rangos (1-10, 11-20, etc.)
M8  Tendencia temporal          → Números en alza/baja en los últimos 30 sorteos
M9  Cadenas de Markov           → Transiciones probables entre sorteos consecutivos
M10 IA Gemini                   → Sugerencia contextual con modelo de lenguaje

NOTAS IMPORTANTES
-----------------
* Las loterías son juegos de azar puro. Ningún sistema puede garantizar
  plenos (5/5). El "acierto >90%" se refiere a tener >=2 números
  correctos en alguna de las 2 jugadas sugeridas, no ganar el premio mayor.
* El sistema se auto-mejora con cada ciclo, ajustando los pesos de los
  métodos según su rendimiento histórico real.
* Los pesos se guardan en data/pesos_metodos.json y persisten entre ejecuciones.
